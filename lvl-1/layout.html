Couple things to point out right now: I can't write up a comprehensive tutorial for you, it'd take years. You'll need to learn how to look up your basic styles. Just google "css [whatever you're trying to do]". But I can go through the basics here:

With color, those that are like #0000FF are called hex colors. They represent RED, BLUE, and GREEN, two digits each in its hex-value, which is like a code for numbers. You'll see letters AND numbers. The only letters are A through F. You also saw "rgba(0,0,0,0.5)". That's also RGB but with an alpha value, making it kinda transparent. That value can be any decimal between 0 and 1, with 0 being totally invisible and 1 being as visible as it gets (no transparency). These color codes can be used to enter color on any color-related thing. Border color, font color, background, etc.

Speaking of border, you saw me use 'border' and 'border-bottom'. You can guess: 'border-left', 'border-right', and 'border-top' are also things too. But 'border' assumes all 4. That also used 3 values in its attribute: A width in pixels (or any unit), a style (solid, dashed, dotted, etc), and a color. Some CSS 'rules' have multiple values, but if they're not specified then defaults will be used.

Similarly you saw three font-names under 'Font Family'. Thats saying "use the first, if it isn't on the user's computer then use the second". That last one "sans-serif" just says "use the computer's standard sans-serif font". If you don't know, serif means the letters have the little flourishes on the tips like Times New Roman does. sans-serif is without those. All fonts can be broken into those two categories, at the very least.

There are a lot of typographic rules involving indents, when/where/if to use an automatic line-break (to "word-wrap" or not), and some other bits. You can google those, we're staying basic.

What about space?

The world of padding, margin, units, and the box-model. Look at the header above: it's moved in on all sides with white space, but it has a background that has yet more space within. The outside of that box is the margin. The inside is the padding. That's it, that's the box model. Memorize it. Margin outside. Padding inside. This seems like a small thing right now, but it factors into literally everything.

Okay, all of this is starting to get a bit difficult on the eyes, right? So far we've been using inline-styles, contained within each element it styles. There's two problems here; One, it sucks to code and read. Two, if you need to re-use the same style over and over AND THEN IT CHANGES, you have to change it everywhere. Pain in the ass. So bottom line, never do it. Instead, use classes.

Way more simple, right? You saw the tag in the head of the document, that's what this is referring to. This element has all of the style rules given by the class named "class-name".

You can name classes anything you want, as you can see. Almost anyway: they can't start with a number, and they cannot have any spaces. When declaring a class in the head, you use a period to indicate "this is a class". When listing a class in an attribute like this paragraph tag, there is no period.

There' also IDs. IDs are kinda different, and rarely should be used for the sake of styling. In the head, you see this is referred to by the hash #anID. Again, that only goes in that style tag, not in the paragraph element. An important part about IDs: You cannot re-use them. Only one element on the page can have a given ID. That's one reason not to use them for styles, there's others I won't get into. General rule: Don't do this.

And finally, we can combine classes this way. This paragraph element has the rules given by both of the classes, we just separate the rule names with a space.
